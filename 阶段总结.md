# 卓软信息工作室社团管理系统计划书与阶段总结

## 一、项目目标

本项目建设一个面向高校社团招新和内部管理的轻量级管理系统，围绕卓软信息工作室的实际招新材料展开。系统包含宣传页面、招新报名系统和后台人员管理三个核心模块，能够完成“了解社团 - 填写报名 - 管理员审核 - 人员跟进”的基本闭环。

## 二、需求分析阶段

已根据《卓软信息工作室报名招新表.docx》提取社团介绍、部门介绍和报名字段。报名表字段包括姓名、性别、出生年月、民族、政治面貌、二级学院、系别、班级、联系电话、QQ、意向部门、是否服从调剂、爱好及特长、曾获奖项、任职经历、职业规划和照片。

系统定位为可直接演示的社团管理系统原型。当前版本采用浏览器本地存储保存报名数据，不依赖服务器和数据库，适合课程展示、前期需求确认和后续二次开发。

## 三、系统规划阶段

系统分为三个页面：

1. `index.html`：宣传页面，展示社团定位、简介、部门介绍和报名入口。
2. `register.html`：招新报名页面，按照 Word 报名表字段设计表单，支持基础校验和提交。
3. `admin.html`：人员管理页面，支持查看报名列表、按部门筛选、按状态筛选、关键词搜索、查看详情和修改审核状态。

公共资源包括：

1. `styles.css`：统一页面样式，形成高校官网风格和管理系统风格。
2. `app.js`：封装报名校验、记录生成、数据筛选、状态修改和页面交互逻辑。
3. `tests/test_app.js`：核心数据逻辑测试文件。

## 四、页面设计阶段

宣传页采用高校官网常见结构，包括顶部导航、首屏招新介绍、社团简介、部门介绍和报名行动区。视觉风格偏正式、清晰和技术社团气质，避免过度娱乐化。

报名页采用分组表单布局，字段与报名表保持一致。必填字段会进行前端校验，手机号和 QQ 号会进行格式校验。提交成功后生成报名编号，并提示用户留意后续通知。

管理页采用普通管理系统布局，包含筛选区、表格区和详情区。管理员可以快速查看报名人基本信息，并通过状态下拉框完成审核流转。

## 五、实现阶段

当前实现选择 `HTML + CSS + JavaScript + localStorage`。这种方式部署简单，双击 `index.html` 即可查看宣传页，也可以通过浏览器直接进入报名页和管理页。

报名数据保存在浏览器 `localStorage` 中。提交报名后，后台管理页可以读取同一浏览器下的报名记录。该方案适合原型演示；如果后续需要多人协作、真实线上部署和长期保存数据，可以升级为 `Node.js + Express + SQLite/MySQL` 版本。

## 八、Vue3 前端迁移阶段

根据新的技术要求，前端已从普通 HTML + JavaScript 多页面结构调整为 Vue3 单页应用结构。最初采用 Vue3 CDN 方式验证页面迁移，随后继续升级为标准 Vite + Vue3 项目，避免依赖 CDN，便于后续接入 Spring Boot 接口。

本阶段完成情况：

1. `index.html` 改为 Vite 入口，通过 `/src/main.js` 挂载 Vue 应用。
2. 新增 `src/App.vue`，将宣传首页、招新报名、人员管理后台整合为三个 Vue 视图。
3. 新增 `src/main.js`、`src/recruitment.js`、`src/recruitment.cjs`，拆分 Vue 启动入口和报名业务逻辑。
4. `register.html` 和 `admin.html` 保留为兼容入口，分别跳转到 `index.html#register` 和 `index.html#admin`。
5. 原报名表字段全部迁移到 Vue 表单中，仍然包含姓名、性别、出生年月、民族、政治面貌、二级学院、系别、班级、联系电话、QQ、意向部门、是否服从调剂、照片、爱好及特长、曾获奖项、任职经历和职业规划。
6. 后台管理功能迁移到 Vue 视图中，继续支持部门筛选、状态筛选、关键词搜索、查看详情和修改审核状态。
7. 后端 Spring Boot 与 MySQL 数据库暂未编写，当前仅在前端使用 `localStorage` 保存演示数据。

测试完成情况：

```bash

npm run build
```

两组测试和 Vite 构建均已通过。当前开发服务器已通过 `http://127.0.0.1:5173/` 打开预览。下一阶段可开始规划 Spring Boot 接口、MySQL 表结构和 Vue 前端接口替换方案。

## 九、前端 UI 与后台入口优化阶段

本阶段完成情况：

1. 顶部公开导航仅保留“宣传首页”和“招新报名”，移除直接显示的“人员管理”入口。
2. 新增“管理员入口”按钮，点击后弹出管理员登录弹窗。
3. 新增前端演示口令校验，口令通过后才能进入 `#admin` 后台视图。当前口令仅用于前端演示，不具备真实安全性。
4. 当用户直接访问 `#admin` 且未通过口令时，会回到宣传页并弹出管理员登录框。
5. 宣传页新增社团能力数据块和招新流程区，让页面更完整，更接近高校官网宣传页。
6. 补充弹窗、管理员入口、招新流程和移动端适配样式。
7. 更新页面测试，确保公开导航不再直接暴露人员管理入口。

测试完成情况：

```bash
npm test
npm run build
```

两项命令均已通过。后续如果接入 Spring Boot，应将当前前端口令替换为真实登录接口、后端会话和角色权限判断。
